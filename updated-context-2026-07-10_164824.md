## Root Signature Architecture

```
mm type (current):
  Param 0: Root 32-bit constants (48 DWORDs, b0)
  Param 1: Root UAV (u0) — source A
  Param 2: Root UAV (u1) — source B
  Param 3: Root UAV (u2) — destination C
  Param 4: Root UAV (u3) — spare (filled with C addr)

Before this session:
  Param 0: CBV (b0) — required ring-buffer alloc + memcpy + GPU VA bind per dispatch
```

## Key Architecture Decisions

1. **Per-lane not block-cooperative dequant**: Simpler code with `WaveReadLaneAt` for scale only, each lane loads its own dword. No extra barriers, no switch/array overhead. Sufficient load reduction (~50% for Q4_0, 8.7× for Q8_0).

2. **Root constants size**: 48 DWORDs chosen to fit within 64 DWORD root sig limit while covering all shader CB sizes. Alternative considered: separate root sig type for GEMV (4 DWORDs) vs other ops — rejected as overly complex.

3. **Harness timing**: Wall-clock `submit_and_wait` timing (not GPU timestamps). Good for relative comparisons but includes CPU recording + submission overhead. Future: add D3D12 timestamp query support for GPU-only timing.

4. **No shader HLSL changes for root constants**: `ConstantBuffer<MMParams> params : register(b0)` works with both CBV and root constants — the D3D12 runtime makes them indistinguishable to the shader.
